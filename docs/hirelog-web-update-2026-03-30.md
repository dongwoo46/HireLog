# HireLog Web 작업 정리 (2026-03-30)

대상: `hirelog-web`

## 완료된 변경
- 비로그인 사용자도 JD 목록/상세 조회 가능
- 비로그인 사용자는 JD 요약 요청 불가(로그인 필요)
- JD 상세에서 비로그인 사용자 조회 전용 UX 적용
- 비로그인 상태 리뷰/준비기록 저장 비활성화
- 서비스 소개 페이지 분리 및 헤더 진입점 정리
- 홈 분기 정책 정리
  - 비로그인: 홈 대신 서비스 소개 + JD 탐색 흐름
  - 로그인: 기존 홈 유지
- 헤더 인증 상태별 액션 정리
  - 비로그인: 로그인/회원가입
  - 로그인: 알림/프로필/로그아웃
- 기존 폰트 유지(신규 폰트 미도입)
- 알림 API 연동 경로 정렬
- 조회/저장 연동(`member-job-summary`, `stages`, `cover-letters`) 보정
- 관리자 JobSummary > JD 수동등록 버튼을 관리자 전용 요청 페이지로 연결
  - `/admin/jd/request`

## JD 상세 리뷰 UX 반영
- 리뷰 작성 버튼 클릭 시에만 작성창 표시
- 1인 1리뷰 정책 프론트 반영
  - 중복 작성 응답 시 작성 상태 잠금 + 안내
- 일반 사용자에게 리뷰 수정/삭제 버튼 비노출
- 익명 작성 토글 정렬/크기 개선
- 점수 입력 UI 개선
  - 기존: 1~10 숫자 버튼
  - 변경: 별점 선택(반별 단위)
  - 규칙: 0.5별 = 1점, 최대 5별 = 10점

## 백엔드 확인(수정 없음)
- 리뷰 점수 스펙: 1~10
- 리뷰 중복 작성 차단: 서비스 검증 + DB unique
- 이번 라운드에서 `hirelog-api` 코드는 수정하지 않음

## 남은 작업 제안
- 관리자 페이지 문자열/상태 라벨 인코딩 깨짐 구간 정리
- 리뷰 목록 필터/정렬 UX 고도화
- 비로그인 전환 CTA 성과 측정 이벤트 정리
- 서비스 소개 페이지 카피 최종 확정
